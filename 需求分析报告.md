## 1. 项目背景与目标

### 1.1 背景

在建材防火检测实验室中，需要对建筑材料样品进行不燃性测试。根据 ISO 11820:2022 标准，需将样品放入加热炉加热至 **750°C**，持续记录 **60 分钟**的温度数据，根据温升和失重率判定材料是否"不燃"。

由于真实试验设备昂贵（单套约 50-80 万元人民币）、耗电量大（峰值功率约 10kW）、单次试验耗时长（含升温和保温约 2 小时），对于教学和培训场景，**使用仿真系统替代真实硬件**具有显著的价值。

### 1.2 目标

开发一套 **Windows 桌面仿真软件**，完整模拟 ISO 11820 不燃性试验的全流程：

- 仿真 5 个温度通道（炉温1、炉温2、表面温、中心温、校准温）的温度变化
- 实现完整的试验状态机（空闲→升温→就绪→记录→完成）
- 实时显示温度曲线和试验数据
- 自动生成 CSV / Excel / PDF 标准格式报告
- 支持试验记录历史查询和设备校准管理

### 1.3 用户群体

| 用户角色 | 说明 | 权限 |
|---------|------|------|
| 管理员 (admin) | 教师/实验室负责人 | 全部功能 + 参数配置 |
| 试验员 (experimenter) | 学生/操作人员 | 执行试验 + 查看记录 |

## 2. 术语定义

| 术语 | 英文 | 说明 |
|------|------|------|
| 不燃性 | Non-combustibility | 材料在高温下不燃烧、不释放可燃气体的特性 |
| 炉温 | Furnace Temperature (TF) | 加热炉内部温度，目标值 750°C |
| 表面温 | Surface Temperature (TS) | 样品表面的温度 |
| 中心温 | Center Temperature (TC) | 样品中心的温度 |
| 失重率 | Mass Loss Rate | 试验后样品质量损失百分比 |
| 温升 | Temperature Rise (ΔT) | 试验结束温度与初始环境温度之差 |
| 恒功率 | Constant Power | PID 控制器输出的稳定加热功率值 |
| 温漂 | Temperature Drift | 单位时间内温度变化的趋势 |
| NCT-Sim | — | 本系统的简称 (Non-combustibility Test Simulator) |

### 3.2 技术栈兼容性分析

```
Windows 10/11 (x64)
    └── .NET 8 Runtime
        ├── Windows Forms (系统原生支持)
        ├── SQLite (通过 Microsoft.Data.Sqlite)
        ├── OxyPlot.WindowsForms (GDI+ 渲染)
        ├── EPPlus (纯托管代码)
        ├── PDFsharp-MigraDoc (纯托管代码)
        └── Serilog.Sinks.File (纯托管代码)
```

**兼容性结论**：全部组件均为纯托管代码或系统原生支持，无需额外安装运行环境，在 Windows 10/11 上可直接运行。

### 3.3 技术难点及解决方案

| 难点 | 描述 | 解决方案 |
|------|------|---------|
| 温度仿真真实性 | 需要仿真 5 通道温度的物理变化过程 | 分阶段仿真算法：升温阶段线性+噪声，稳定阶段钳位+噪声，记录阶段指数逼近模型 |
| 实时曲线渲染 | 每秒更新 4 条曲线的海量数据点 | OxyPlot 增量更新，X 轴滚动窗口（最近 10 分钟），避免全量重绘 |
| 跨线程 UI 更新 | 后台采集线程需更新 UI 控件 | 事件驱动 + `Control.Invoke()` 封送至 UI 线程 |
| 状态机管理 | 5 状态 + 多条件转换 + 并发安全 | 集中式状态控制器 + `lock` 同步 |
| 报告生成 | Excel 含嵌入图表，PDF 含曲线图片 | EPPlus 内嵌图表 API + PDFsharp 位图嵌入 |

### 3.4 技术可行性结论

> **方案成熟可靠**，全部采用业界主流、文档丰富、社区活跃的技术栈。核心技术难点（实时仿真、状态机、报告生成）均有成熟解决方案，不存在不可逾越的技术障碍。

## 5. 操作可行性分析

### 5.1 用户学习成本评估

| 用户类型 | 所需技能 | 学习时间 | 说明 |
|---------|---------|:---:|------|
| 管理员 | 基础 Windows 操作 | < 30 分钟 | 图形界面引导，无需命令行 |
| 试验员 | 基础计算机使用 | < 15 分钟 | 流程引导式操作 |

### 5.2 操作流程简化设计

系统采用**向导式流程**，操作步骤与真实试验完全一致：

```
启动程序 → 选择角色 → 输入密码
    ↓
新建试验 → 填写样品信息 → 确认创建
    ↓
开始升温 → 等待温度稳定（自动判定）
    ↓
开始记录 → 等待试验完成（自动计时）
    ↓
填写试验记录 → 保存 → 自动生成报告
```

每一步都有明确的状态提示和按钮引导，用户无需记忆操作流程。

### 5.3 与用户习惯匹配度

| 设计特点 | 匹配的用户习惯 |
|---------|--------------|
| 标准 Windows 窗体 | 与 Office 等常用软件一致 |
| 菜单栏 + 工具栏 | 经典桌面应用布局 |
| Tab 页切换 | 类似浏览器标签页 |
| 大字体 LED 风格温度显示 | 模拟真实仪表盘 |
| 右键上下文菜单 | Windows 标准交互 |
| 快捷键支持 | Enter 确认、Esc 取消 |

### 5.4 操作可行性结论

> **用户体验无缝衔接**。界面设计遵循 Windows 桌面应用标准规范，操作流程与真实试验一致，学习成本极低。
