## 2. 核心模块详细设计

### 2.1 仿真引擎 (SensorSimulator)

#### 算法输入

| 参数 | 来源 | 默认值 |
|------|------|:---:|
| TargetFurnaceTemp | appsettings.json | 750.0°C |
| InitialFurnaceTemp | appsettings.json | 720.0°C |
| HeatingRatePerSecond | appsettings.json | 40.0°C/s |
| TempFluctuation | appsettings.json | 0.5°C |
| StableThreshold | appsettings.json | 3.0°C |

#### 升温阶段算法（TF1 < 747°C）

```
每次 tick (800ms):
  TF1 += HeatingRatePerSecond × 0.8 + Random(-1,1) × TempFluctuation
  TF2 += HeatingRatePerSecond × 0.8 + Random(-1,1) × TempFluctuation  (独立噪声)
  TS  = TF1 × 0.3 + Random(-1,1) × TempFluctuation
  TC  = TF1 × 0.25 + Random(-1,1) × TempFluctuation
  TCal = TF1 + Random(-1,1) × TempFluctuation × 2
```

#### 稳定阶段算法（TF1 ≥ 747°C）

```
每次 tick:
  TF1 = 750.0 + Random(-1,1) × TempFluctuation  (钳位到目标)
  TF2 = 750.0 + Random(-1,1) × TempFluctuation
  TS  = TF1 × 0.3 + Random(-1,1) × TempFluctuation
  TC  = TF1 × 0.25 + Random(-1,1) × TempFluctuation
  TCal = TF1 + Random(-1,1) × TempFluctuation × 2

  StableCounter++
  IF StableCounter > 3 AND 745 ≤ TF1 ≤ 755 THEN
    IsStable = true
    → 状态切换: Preparing → Ready
```

#### 记录阶段算法

```
每次 tick:
  // 炉温保持
  TF1 = 750.0 + Random(-1,1) × TempFluctuation
  TF2 = 750.0 + Random(-1,1) × TempFluctuation

  // 样品温度指数逼近
  surfaceTarget = min(TF1 × 0.95, 800.0)
  TS += (surfaceTarget - TS) × 0.02 + Random(-1,1) × TempFluctuation

  centerTarget = min(TF1 × 0.85, 750.0)
  TC += (centerTarget - TC) × 0.01 + Random(-1,1) × TempFluctuation

  TCal = TF1 + Random(-1,1) × TempFluctuation × 2
```

#### 降温阶段算法

```
每次 tick:
  TF1 -= 0.5 + Random(-1,1) × TempFluctuation × 0.1
  TF2 -= 0.5 + Random(-1,1) × TempFluctuation × 0.1
```

#### 状态转换规则表

| 当前状态 | 触发条件 | 目标状态 | 动作 |
|---------|---------|:---:|------|
| Idle | 用户点击"开始升温" | Preparing | 初始化仿真引擎 |
| Preparing | TF1∈[745,755] 且稳定计数>3 | Ready | 计算恒功率值 |
| Ready | 用户点击"开始记录" | Recording | 开始计时，写入CSV |
| Ready | TF1 跌出 [745,755] | Preparing | 继续等待稳定 |
| Recording | 到达设定时长/终止条件 | Complete | 停止计时 |
| Recording | 用户点击"停止记录" | Complete | 停止计时 |
| Complete | 用户保存试验记录 | Preparing | 保持炉温，更新数据库 |
| Preparing | 用户点击"停止升温" | Idle | 开始降温 |
| Ready | 用户点击"停止升温" | Idle | 开始降温 |
| Complete | 用户点击"停止升温" | Idle | 开始降温 |

### 2.3 温漂计算 (TemperatureAnalyzer)

使用 **MathNet.Numerics** 的线性回归计算：

```csharp
// 对最近 10 分钟（600 个数据点）的炉温做线性回归
var regression = new SimpleRegression();
for (int i = history.Count - 600; i < history.Count; i++)
{
    regression.Add(i, history[i].TF1);
}
double drift = regression.Slope * 600; // 转换为 °C/10min

// 判定稳定条件
bool isStable = Math.Abs(drift) < 2.0; // 温漂 < 2°C/10min
```

### 2.4 试验终止条件

```
标准 60 分钟模式:
  ├── 每 5 分钟检查（30, 35, 40, 45, 50, 55 分钟）
  │   └── 条件：10分钟温漂有效 且 炉温1/炉温2的10分钟温漂 ≤ MaxTemperatureDriftPerTenMinutes
  └── 第 60 分钟（3600秒）无条件终止

固定时长模式:
  └── 忽略提前终止，到达 TargetDurationSeconds 后完成

手动终止:
  └── 用户点击"停止记录"
```

## 5. 异常处理设计

### 5.1 异常处理层次

```
┌─────────────────────────────────┐
│        UI 层异常处理             │
│  try-catch → MessageBox 提示    │
│  记录到 Serilog 日志            │
└─────────────────────────────────┘
              ↑
┌─────────────────────────────────┐
│       服务层异常处理             │
│  抛出有意义的业务异常            │
│  包含上下文信息                  │
└─────────────────────────────────┘
              ↑
┌─────────────────────────────────┐
│       数据层异常处理             │
│  数据库错误 → SqliteException   │
│  文件IO错误 → IOException       │
└─────────────────────────────────┘
```

### 5.2 关键异常场景处理

| 场景 | 异常类型 | 处理方式 |
|------|---------|---------|
| 数据库连接失败 | SqliteException | 提示用户检查数据库文件路径 |
| CSV 写入失败 | IOException | 提示磁盘空间不足，停止记录 |
| 配置文件缺失 | FileNotFoundException | 使用默认值，警告日志 |
| 跨线程 UI 操作 | InvalidOperationException | 开发阶段断言检查，防止发生 |
| PDF 生成失败 | Exception | 提示导出失败，建议重试 |

### 5.3 容错机制

```
数据库操作:
  ├── using 语句确保连接释放
  ├── 事务保护批量写入
  └── 失败重试 1 次

文件操作:
  ├── 写入前检查目录是否存在，不存在则创建
  ├── StreamWriter 追加模式
  └── 写入失败不影响主流程

仿真引擎:
  ├── 参数范围校验（防止除零、负数温度）
  └── 异常时使用上次有效值
```
